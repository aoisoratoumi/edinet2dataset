# 企業リサーチレポート（EDINET × PostgreSQL）

MCP サーバー `edinet-postgres` のツールを使い、有価証券報告書データから対象企業の
**過去の経営成績・経営戦略・戦術** を調査してレポートにまとめる。

## 前提条件

- MCP サーバー `edinet-postgres` が接続されていること（`.mcp.json` で登録済み。ツールが見えない場合はユーザーに `/mcp` での接続確認を依頼する）。
- PostgreSQL にデータが投入済みであること。データが古い・無い場合は `/edinet-download` スキルで最新化できることを案内する。

## 調査手順

### 1. 企業の特定

`search_company` で企業名から EDINET コードを特定する。

- **ヒットしない場合は表記ゆれを試すこと**: カタカナ⇔英語⇔漢字、「株式会社」の有無など。
  例: 「サイバーダイン」→ `CYBERDYNE`、「トヨタ」→ `トヨタ自動車`。
- それでも見つからなければ `run_readonly_sql` で `meta->>'会社名'` も含めて検索する。
- 複数ヒットしたら決算期の範囲やレポート件数からユーザーの意図する企業を選ぶ（曖昧なら確認する）。

続けて `list_reports` で保存されている決算期の範囲を把握する。

### 2. 経営成績の調査（定量）

`get_financial_history` で主要指標の推移を取得する。

- 金額は円単位。レポートでは億円などに換算して読みやすくする。
- **利益は売上高・営業利益だけで終わらせず、経常利益・（親会社株主に帰属する）当期純利益まで必ず並べて追う**こと。
  経常利益は本業に加え金融収支など経常的な収益力を示す指標であり、営業利益との差（営業外損益）や
  当期純利益との差（特別損益・税金）から一過性要因の有無を読み取る。
- その他の見るべきポイント: 各利益の成長トレンド・利益率の変化、
  ROE・自己資本比率、営業CFと投資CFのバランス（成長投資フェーズか回収フェーズか）、
  従業員数の増減、配当政策。
- 「最古レポートの5期推移」を使えば保存レポートより前の年度もカバーできる。
- IFRS 企業は summary に無い指標がある（売上高は「売上高（PL: 営業収益合計）」等として PL から補完される）。
  それでも欠ける指標は `run_readonly_sql` で bs / pl / cf JSONB を直接引く。

### 3. 経営戦略・戦術の調査（定性）

`list_text_sections` でセクションを確認し、`get_text_section` で以下を読む:

| セクション | 読み取る内容 |
|---|---|
| 事業の内容 | ビジネスモデル・セグメント構成 |
| 経営方針、経営環境及び対処すべき課題等 | **経営戦略の中心**。中期目標・重点施策・KPI |
| 財政状態、経営成績及びキャッシュ・フローの状況の分析 | 業績変動の要因（MD&A）・戦術の実行結果 |
| 研究開発活動 | 技術戦略・開発投資の方向性（戦術） |
| 事業等のリスク | 経営環境の制約・弱み |

- 長いセクションは `offset` / `limit` でページングして読む。
- **複数年度分を取得して比較する**こと（`fiscal_year_end` で年度指定）。
  戦略の変遷（何を掲げ、何を達成/撤回したか）はレポートの核になる。

### 4. レポート作成

以下の構成でまとめる:

1. **会社概要** — 社名・EDINETコード・証券コード・事業内容の要約
2. **経営成績の推移** — 決算期ごとに **売上高・営業利益・経常利益・（親会社株主に帰属する）当期純利益** を
   並べた表（億円換算）を必ず作成し、複数社を扱う場合はサマリー表にも経常利益の列を含める。
   ROE・自己資本比率等の表 + トレンドの解説を続ける。
3. **経営戦略** — 各年度の有報で掲げられた方針・中期目標とその変遷
4. **戦術・具体的施策** — 研究開発・設備投資・MD&A から読み取れる実行策と成果
5. **リスクと課題** — 事業等のリスクの要約
6. **総括** — 定量（成績）と定性（戦略）を突き合わせた評価

記載する数値には必ず決算期を明記する。データの出典が有価証券報告書（EDINET）であること、
DB に入っている決算期の範囲をレポート冒頭か末尾に記すこと。

## 注意

- DB に無い年度や企業のデータは推測で補わない。不足していれば「DB 未収録」と明記し、
  必要なら `/edinet-download` でのデータ取得を提案する。
- `run_readonly_sql` は SELECT 専用。同業他社との比較集計などに活用できる。
